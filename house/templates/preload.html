SideLoad Stetho

Stetho is a debug bridge for Android applications that I find it to be very useful, House can sideload stethor.jar and inject it into your application for debug purpose. It will only be turned on if you click the button below.

Use with caution: It is experimental for now as Stetho requires android.support.v4 support, disable Stetho if the app crashes with "java.lang.ClassNotFoundException".

Here is what you need to do to make it work:

  • 1. Download Stetho stand-alone JAR.
  • 2. Rename the download to stetho.jar.
  • 2.1 Convert the jar file to dex, using tool such as dextojar d2j-jar2dex.sh stetho.jar.
  • 3. Push the converted-dex file to the device under this diretory: /data/local/tmp/stetho.jar, e.g. adb push stetho-jar2dex.dex /data/local/tmp/stetho.jar.
  • 4. Click the Load button below, note this would restart the application.
  • 5. To unload stetho.jar, simply restart the app.
  • 6. What's new on chrome://inspect/#devices?

SSL Unpinning

Toggle the button on the right-hand side to preload SSL Unpinning frida script.

Here is what you need to do to make it work:

  • 1. Export your proxy certification, e.g. burpca-cert-der.crt
  • 2. Save the cert on the device as: /data/local/tmp/cert-der.crt;
    E.g. adb push burpca-cert-der.crt /data/local/tmp/cert-der.crt.

Proxy Setting (Current app only)

House support proxy over adb reverse binding, all app traffic will go through 127.0.0.1:8080 on the device; run adb reverse tcp:8080 tcp:8080 to bind it to the proxy host
